# 情報セキュリティマネジメントの学習記録
## 2026.05.27
### 今日の学び
・リスクマネジメントは特定、分析、評価、対応の順番で、見つける、理解する、順位づけ、実施のイメージ
・SIEMはログを集中管理する製品。インシデントを検知、分析、可視化する。
・ゼロトラストは内部も安全じゃないぞ！という考え方。トラストは信頼という意味
・不正のトライアングルは動、機、正。同期生て覚える。
・DNSキャッシュポイズニングはDNSをすり替えて誤ったwebに誘導すること。対策としてDNSSEC(デジタル証明書を使うやつ)や、外部からの再起的な問い合わせを受け付けないようにする

## 2026.05.28
### 今日の学び
・CVSSは基本、現状、環境に分かれていて、基本はベースのチェック、現状は今の状態チェック、環境は利用者の二次被害チェック
・特定電子メール法は営利団体の送信者又は送信委託者が勝手に顧客へメール出すなよという法
・懲戒手続は疑いの段階で実施しちゃダメ。冤罪もありうるから
・RASISは、システムが故障しないで動く事を見る信頼性、システムが通常通り動いているか見る可用性、システムが復旧するまでの時間を見る保守性、障害が起きてもデータが壊れてないか見る完全性、データが安全に守れているか見る機密性がある
・データマートはデータウェアハウスから取り出して加工したデータ
・デジタルトランスフォーメーションには3段階あり、デジタイゼーションは単純なデジタル化、デジタライゼーションは個別業務やプロセスのデジタル化、デジタルトランスフォーメーションは全社的な業務やプロセスのデジタル化

## 2026.06.04
### 今日の学び
・可用性はいつでも通常通り
・メールの送信ミスはどう防ぐか
・匿名加工情報は特定できたらダメ、復元できてもダメ
・リスクマネジメントは特定、分析、評価
・言い訳させないためにデータにはラベル付けする
・コンペアは比較、チェックサムは事前に保証、パターンマッチは検索、ビヘイビアはマルウェア実行
・サイバーキルチェーンは攻撃者の行動モデル

## 2026.06.04
### 今日の学び
・リスクベース認証はいつもと違うアクセスだよと教えてくれる
・サーバー証明書はCAの秘密鍵でデジタル署名して、受取人は公開鍵で正しさ検証
・個人情報が保護されるのは、生きてる人だけ
・内部統制の統制活動はトップの命令が結果に反映されてるか検証
・RASISのAは可用性、MTBFを大きく、MTTRを小さく
・プロキシサーバはPCの代わりに外部接続
・RPAはルールが出来てる定例業務が有効
・特性要因図はフィッシュボーン原因と結果の因果を知る
・偽セキュリティ警告はサポート詐欺ともいう、セキュリティソフトがアラームを出してるように見せる騙し方
